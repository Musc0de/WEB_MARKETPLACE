# Design QA and Handoff

Setiap page handoff harus menyertakan:

- route
- role/access
- desktop dan mobile viewport
- component inventory
- token yang dipakai
- states
- API/data assumptions
- toast mapping
- accessibility notes
- analytics events
- acceptance criteria

## Review order

1. Functional completeness.
2. Responsive composition.
3. Content clarity.
4. Interaction states.
5. Accessibility.
6. Visual consistency.
7. Performance/layout shift.

Screenshot saja bukan acceptance evidence; sertakan hasil interaction dan state test.
