# Page Specification Template

Gunakan struktur berikut untuk halaman baru:

1. Tujuan pengguna.
2. Route dan access level.
3. Entry/exit points.
4. Desktop composition.
5. Mobile composition.
6. Data dependency.
7. Component list.
8. Interaction and validation.
9. Loading/empty/error/offline states.
10. Gooey Toast mapping.
11. Accessibility.
12. Analytics events.
13. Acceptance criteria.
14. Proposed source file map.

Jangan menulis hanya screenshot description. Dokumentasi harus menjelaskan behavior dan state.
